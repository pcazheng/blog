HBase

HBase简介

高可靠，高性能，面向列，可伸缩，实时读写的分布式数据库
利用HDFS作为其文件存储系统，支持MR程序读取数据
存储非结构化和半结构化数据

##RowKey
数据唯一标示，按字典排序

##Column Family
列族，多个列的集合 最多不要超过3个 5个
列簇里面的列没有限制 动态增加 只有插入列数据才存在
列在列簇中时有序的

##TimeStamp时间戳
支持多版本数据同时存在

#应用场景
海量数据存储 准实时查询
可考虑 百度统计数据

##大容量
HBase单表可以有百亿行，百万列，数据矩阵横向和纵向两个维度所支持的数据量级都非常具有弹性

##面向列
HBase是面向列的存储和权限控制，并支持独立检索。列式存储，其数据在表中是按照某列存储的，这样在查询只需要少数几个字段的时候，能大大减少读取的数据量
稀疏性 为空的列并不占用存储空间

##多版本
HBase每一个列的数据存储都有多个version（可储存历史）

##扩展性
底层依赖HDFS 高可靠性 WAL机制保证了数据写入时不会因集群异常而导致写入数据丢失：Replication机制保证了在集群出现严重的问题时，数据不会发生丢失或损坏。而且HDFS底层有备份
底层的LSM数据结构和rowKey有序排列等架构的独特设计，似的HBase具有非常高的写入性能。region切分、主键索引和缓存机制使得Hbase在海量数据下具备一定的随机读取性能，该性能针对RowKey的查询能够达到毫秒级
官网http://hbase.apache.org

##不支持条件查询
只支持RowKey查询

Hadoop

大数据、原理与机制
开源的大数据框架、分布式计算解决方案
HDFS（分布式文件系统）+MapReduce（分布式计算）

#HDFS
HDFS =>数据块 & NameNode & DataNode

##数据块
数据块是抽象块而非真个文件作为存储单元，默认大小为64MB，一般设置为128MB，备份x3

##NameNode
管理文件系统的命名空间，存放文件元数据 尾货着文件系统的所有文件和目录，文件与数据库的映射 记录每个文件中各个块所在数据节点的信息（主）

##DataNode
存储病建设数据块 向NameNode更新所存储块的列表
优点 适合大文件存储，支持TB、PB的数据存储，并有副本策略 可以构建在廉价的机器上，并有一定的容错和恢复机制 可以支持流式数据访问，一次写入，多次读取最高效
缺点 不适合大量小文件存储 不适合并发写入，不支持文件随机修改 不支持随机读等低延时的访问方式
主节点+备用节点 避免单点NamNode挂掉无法响应问题–

#MapReduct
编程模型，是一种编程方法，是抽象理论
输入一个大文件，通过split之后，将其分为多个分片
每个文件分片由单独的机器去处理，这就是Map方法
将各个机器计算的结果进行汇总并得到最终的结果，这就是Reduce方法
Map+Reduce=>Stream+Collect
自定义Map方法和Reduce 对HDFS里面的数据进行处理输出

##YARN 资源管理器

###ResourceManager
分配和调度资源 启动并监控Application 监控NodeManager

###ApplicationMaster
为MR类型的程序申请资源，并分配给内部任务 负责数据的切分 监控任务的执行及容错

###NodeManager
管理单个节点的资源 处理来自ResourceManager的命令 处理来自ApplicationMaster的命令

docker安装流程

#下载安装docker for docker
地址：http://www.docker.com

#配置阿里云加速

#常用命令
docker images 打印本地所有镜像
docker run -p 8080:80 -d daocloud.io.ngix
 -p 本地端口:服务器端口 端口映射
 -d 直接返回 服务作为守护进程运行
docker cp index.html dockeID://路径 拷贝文件到指定容器
docker stop dockerID 停止容器服务
docker commit -m ‘fun’ dockerID 新镜像名称
docker rmi 镜像ID 删除镜像
docker rm 容器ID 删除容器
docker ps 当前运行的容器 -a列出所有容器运行中跟停止的
docker pull 获取image
docker build 创建image
docker exec -it nginx /bin/base 进入容器 ctrl+d退出
docker inspect nginx 检查打印容器信息
docker search nginx 搜索镜像
docker pull nginx 下载镜像
docker push my/nginx 上传镜像
docker login //登录

#通过dockerfile创建镜像
vi mydockerfile
FROM ubuntu //基础镜像
MAINTAINER pcaz //镜像作者
RUN sed i ‘s/archive.ubuntu.com/mirrors.ustc.edu.cn/g’ ‘etc/apt/sources.list’//使用国内加速镜像仓库
RUN apt-get update //更新ubuntu
RUN apt-get install -y nginx //安装ngix -y：不要询问
COPY index.html /var/www/html
ENTRYPOINT [“/usr/sbin/ngix”,”-g”,”daemon off;”]//将nginx作为前台执行而不是守护进程执行 字符串数组会自动空格隔开运行
EXPOSE 80//暴露80端口
docker build -t 文件名 . //最后点表示当前目录作为上下文

#容器特点
分层 用dockerfile创建的每行命令都会产生一个id 相同id共享避免镜像过大
Volume 提供独立于容器之外的持久化存储共享
docker run -p 80:80 -d -v $PWD/html:/usr/share/nginx/html nginx
-v 挂载 $PWD/html当前目录html文件挂载到容器/usr/share/nginx/html
